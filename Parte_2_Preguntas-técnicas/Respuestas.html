Parte 2 – Preguntas técnicas

1. ¿Qué ventajas encuentras en usar Angular frente a otros frameworks?

Angular proporciona una arquitectura escalable y robusta, integración nativa con TypeScript, un sistema de inyección de dependencias avanzado y herramientas integradas para testing, routing y gestión de estados, lo que permite desarrollar aplicaciones empresariales mantenibles y seguras.

2. ¿Qué estructura usás al desarrollar un backend en PHP y por qué?

Prefiero una arquitectura MVC desacoplada, aplicando principios SOLID y separación de responsabilidades, lo que facilita la escalabilidad, el testing automatizado y el mantenimiento eficiente del código en equipos colaborativos.

3. ¿Cómo aplicarías buenas prácticas de CI/CD en un equipo ágil?

Implemento pipelines automatizados con integración y despliegue continuo, asegurando validaciones automáticas, pruebas unitarias y despliegues controlados, para minimizar errores humanos y acelerar la entrega de valor al usuario final.

4. ¿Cómo manejarías un bug crítico en producción que afecta a todos los usuarios?

Priorizaría la comunicación inmediata, aislaría el bug mediante rollback o hotfix, desplegaría la solución lo antes posible y luego realizaría un análisis post-mortem para fortalecer la prevención y monitoreo futuro.

5. ¿Qué consideraciones tomás para optimizar una consulta MySQL con muchos registros?

Analizo índices, evito operaciones costosas como subconsultas innecesarias, aplico paginación, reviso el plan de ejecución y limito los campos seleccionados para maximizar la eficiencia y reducir la latencia.

6. ¿Has usado servicios en la nube como AWS? ¿Cuáles y para qué?

Sí, he utilizado EC2 para infraestructura, S3 para almacenamiento escalable, RDS para bases de datos gestionadas y Lambda para funciones serverless, optimizando despliegues, disponibilidad y automatización de procesos.
